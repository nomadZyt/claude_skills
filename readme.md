# Weekly Report Skill

一个用于自动生成项目周报的 Claude Skill，可以分析 Git 提交记录并生成结构化的周报文档。

## 前置要求

- 已安装 Claude Code（Claude Desktop 应用）

## 安装步骤

1. **创建工作目录**
   - 在任意位置新建一个文件夹作为工作目录

2. **初始化 Claude 配置**
   - 进入该目录，执行 `claude` 命令
   - 完成初始化后，会在当前目录生成 `.claude` 文件夹

3. **安装 Skill**
   - 在 `.claude` 目录下创建 `skills` 文件夹（如果不存在）
   - 将 `weekly-report-skill` 目录复制到 `.claude/skills/` 目录下

4. **加载 Skill**
   - 执行 `claude` 命令
   - 输入：`加载 weekly-report-skill`

## 使用方法

执行 `claude` 命令后，输入以下指令：

```
使用weekly-report-skill对[项目路径]生成周报
```

**参数说明：**
- `[项目路径]`：可以是项目的根目录路径，或包含 Git 仓库的文件夹路径
- 支持绝对路径和相对路径

**使用示例：**
```
使用weekly-report-skill对/Users/user/Projects/my-project生成周报
```

或

```
使用weekly-report-skill对./my-project生成周报
```

## 功能说明

该 Skill 会自动：
- 分析指定项目的 Git 提交记录
- 按时间范围分类提交内容
- 生成结构化的周报文档


# project-analyzer

一个用于系统性分析项目结构、技术栈、数据流转、核心功能和业务特性的 Claude Skill，支持全量分析和模块分析两种模式。

## 前置要求

- 已安装 Claude Code（Claude Desktop 应用）

## 安装步骤

1. **创建工作目录**
   - 在任意位置新建一个文件夹作为工作目录

2. **初始化 Claude 配置**
   - 进入该目录，执行 `claude` 命令
   - 完成初始化后，会在当前目录生成 `.claude` 文件夹

3. **安装 Skill**
   - 在 `.claude` 目录下创建 `skills` 文件夹（如果不存在）
   - 将 `project-analyzer` 目录复制到 `.claude/skills/` 目录下

4. **加载 Skill**
   - 执行 `claude` 命令
   - 输入：`加载 project-analyzer`

## 使用方法

执行 `claude` 命令后，可以使用以下两种模式：

### 模式1：全量分析
输入以下指令进行全量项目分析：
```
帮我梳理一下这个项目
```
或
```
分析整个项目结构
```
或
```
生成项目分析文档
```

### 模式2：模块分析
输入以下指令分析特定功能模块：
```
分析一下 [模块名称] 功能
```
或
```
梳理 [模块名称] 模块
```
或
```
[模块名称] 是怎么实现的
```

**使用示例：**
```
分析一下用户登录功能
```
或
```
梳理订单管理模块
```

## 功能说明

该 Skill 会自动进行六维度分析：

1. **项目概览**：一句话总结项目定位、核心业务场景和目标用户
2. **技术栈识别**：列出项目使用的所有技术及版本
3. **项目结构分析**：理清代码组织方式和模块职责
4. **数据流转分析**：追踪数据从API到UI的完整链路
5. **核心功能实现**：梳理关键业务流程的代码实现
6. **业务特性**：识别项目的业务规则和特殊逻辑

**适用场景：**
- 技术交接
- 代码审查
- 项目复盘
- 新人 onboarding
- 功能模块理解
