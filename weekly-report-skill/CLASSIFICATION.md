# 工作类型分类规则

## 1. 需求开发

定义：
- 直接交付业务能力或用户功能

常见信号：
- commit 包含：feat / feature / 新增 / 支持
- 涉及页面、交互、接口新增

---

## 2. Bug 修复

定义：
- 修复已知问题或线上缺陷

常见信号：
- fix / bug / 修复 / 问题
- 回滚、补丁类提交

---

## 3. 技术债 / 重构 / 优化

定义：
- 不直接改变业务能力
- 提升代码质量、结构或性能

常见信号：
- refactor / optimize / clean
- 目录调整、逻辑拆分

---

## 4. 支撑性工作

定义：
- 为业务或团队提供保障，但不直接交付功能

常见信号：
- 配置、CI、脚手架
- 依赖升级、工具调整

---

## 5. 协作 / 合并

定义：
- 非主要产出型工作

处理规则：
- merge / sync / chore
- 默认弱化描述或合并展示
